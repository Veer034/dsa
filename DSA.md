## Daily Routine (1.5-2 hours)

### Efficient Problem-Solving:

* [ ] **Read problem (3 mins):** Understand clearly
* [ ] **Think (5 mins):** Brute force → Optimize
* [ ] **Code (15-20 mins):** Write clean solution
* [ ] **Test (5 mins):** Edge cases
* [ ] **Review (5 mins):** Alternative approaches

**If stuck after 15 mins:** Look at hints/solution, understand pattern

## Pattern Recognition (The Real Skill)

### When you see these keywords:

* [ ] **"Subarray", "substring"** → Sliding Window or Hash Map
* [ ] **"All permutations/combinations"** → Backtracking
* [ ] **"Minimum/Maximum in path"** → DP or DFS
* [ ] **"Connected components"** → Union Find or DFS
* [ ] **"K largest/smallest"** → Heap
* [ ] **"Sorted array search"** → Binary Search
* [ ] **"Graph dependencies"** → Topological Sort
* [ ] **"Shortest path"** → BFS or Dijkstra
